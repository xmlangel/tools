## 애플리케이션 시작하기

### Docker Compose 실행

```bash
# 프로젝트 루트 디렉토리에서
docker-compose up --build

# 백그라운드 실행
docker-compose up -d --build
```

### 서비스 접속

애플리케이션이 시작되면 다음 서비스에 접근할 수 있습니다:

| 서비스 | URL | 설명 |
|--------|-----|------|
| Frontend | http://localhost:5173 | React 웹 인터페이스 |
| Backend API | http://localhost:8000 | FastAPI 백엔드 |
| API 문서 | http://localhost:8000/docs | Swagger UI |
| MinIO Console | http://localhost:9003 | 객체 스토리지 관리 |
| PostgreSQL | localhost:5436 | 데이터베이스 (외부 접속용) |

### 서비스 중지

```bash
# 서비스 중지 (컨테이너 유지)
docker-compose stop

# 서비스 중지 및 컨테이너 삭제
docker-compose down

# 볼륨까지 삭제 (데이터 초기화)
docker-compose down -v
```

## 사용자 인터페이스

### 로그인

1. 브라우저에서 http://localhost:5173 접속
2. 개발 환경 기본 계정으로 로그인:
   - **이메일**: `admin@example.com`
   - **비밀번호**: `admin123`

### 메인 대시보드

로그인 후 메인 대시보드에서 다음 기능에 접근할 수 있습니다:

- **작업 목록**: 진행 중/완료된 작업 확인
- **설정**: 사용자 프로필 및 환경 설정

## YouTube STT 기능

### 기본 사용 흐름

1. **YouTube URL 입력**
   - "YouTube STT" 메뉴 클릭
   - YouTube 동영상 URL 입력
   - 예: `https://youtu.be/dQw4w9WgXcQ`

2. **모델 선택**
   - Whisper 모델 선택 (tiny, base, small, medium, large)
   - 모델이 클수록 정확도 높지만 처리 시간 증가

3. **옵션 설정**
   - **언어**: 자동 감지 또는 수동 선택
   - **출력 형식**: TXT, SRT, VTT, JSON
   - **타임스탬프**: 포함 여부 선택

4. **변환 시작**
   - "변환 시작" 버튼 클릭
   - 작업이 백그라운드에서 실행됨

5. **결과 확인**
   - 작업 목록 페이지로 자동 이동
   - 진행 상황 실시간 모니터링
   - 완료 후 결과 다운로드

## 파일 업로드 기능

### 지원 파일 형식

- **오디오**: mp3, m4a, wav, flac, aac, ogg
- **동영상**: mp4, avi, mkv, mov (오디오 트랙 추출)

### 파일 크기 제한

- 최대 파일 크기: **500MB**
- 더 큰 파일은 CLI 도구 사용 권장

### 업로드 방법

#### 방법 1: 드래그 앤 드롭

```
┌─────────────────────────────────────┐
│                                     │
│         🎵                          │
│                                     │
│   파일을 여기에 드롭하세요          │
│                                     │
│   또는 클릭해서 선택                │
│                                     │
│   지원 형식: mp3, m4a, wav, etc.   │
│   최대 크기: 500MB                  │
│                                     │
└─────────────────────────────────────┘
```

#### 방법 2: 파일 선택

1. "파일 선택" 버튼 클릭
2. 파일 탐색기에서 오디오 파일 선택
3. "열기" 클릭

### 업로드 후 처리

1. 파일 업로드 완료 대기
2. Whisper 모델 선택
3. 옵션 설정
4. "변환 시작" 버튼 클릭

### 업로드 진행 상황

```
┌─────────────────────────────────────┐
│ 파일 업로드 중...                   │
├─────────────────────────────────────┤

│ 📁 audio_file.m4a                  │
│ 크기: 45.2 MB / 120.5 MB            │
│                                     │
│ [████████████────────] 75%          │
│                                     │
│ 업로드 속도: 2.5 MB/s               │
│ 남은 시간: 30초                     │
│                                     │
│         [취소]                      │
│                                     │
└─────────────────────────────────────┘
```

## 💡 사용 팁

### 1. 긴 동영상 처리

- 1시간 이상 동영상은 small 이하 모델 권장
- 백그라운드에서 실행되므로 브라우저 닫아도 됨
- 완료 시 이메일 알림 설정 권장

### 2. 배치 처리

- 여러 파일 동시 업로드 가능
- 작업 큐에서 순차적으로 처리됨
- 우선순위 설정 가능 (프리미엄 기능)

### 3. 결과 공유

- 완료된 작업 공유 링크 생성 가능
- 24시간 동안 유효한 임시 링크
- 다운로드 횟수 제한 설정 가능

### 4. 모바일 사용

- 반응형 디자인으로 모바일 지원
- 파일 업로드 및 작업 확인 가능
- 무거운 처리는 PC 사용 권장

## 🚨 주의사항

### 개인정보 보호

- 업로드된 파일은 서버에 저장됨
- 민감한 내용은 로컬 CLI 도구 사용 권장
- 완료된 작업은 정기적으로 삭제 권장

### 저작권

- YouTube 동영상 저작권 확인 필수
- 개인 학습/연구 목적으로만 사용
- 상업적 이용 시 저작권자 허가 필요

### 서비스 제한

- 동시 처리 작업: 최대 3개
- 일일 처리 한도: 20개 작업
- 저장 공간: 10GB (기본)
